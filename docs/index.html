Why github-pick凭什么用它

It judges by reception, not by birthday.它看的是反响，不是出生日期。

"Awesome lists" go stale and star counts alone lie. github-pick asks one question per project — what is its real-world reception right now? — grounded in velocity, recent activity, discussion, and integrations, then lays it out as a board you can actually scan.awesome 清单会过期，光看星数也会骗人。github-pick 对每个项目只问一句——它现在的真实反响如何？——以增速、近期活跃、讨论度、被集成情况为依据，再铺成一块你真能扫读的看板。

A1
Broad search, then judgment先广搜，再判断

It casts a wide net across GitHub, then scores each candidate on real signals — stars + velocity, recent activity, newsletter / awesome-list mentions, HN/Reddit/知乎 discussion, integrations — verified with web search, not vibes.它在 GitHub 上广撒网，再用真实信号给每个候选打分——星数+增速、近期活跃、newsletter / awesome 清单提及、HN/Reddit/知乎 讨论、被集成情况——靠网页搜索核实，不靠感觉。

A2
Reception, not birthday反响，不是生日

A 2019 project and a 2026 one are both eligible. github-pick never pre-filters by creation date — age is expressed afterward through tags, never used to exclude up front.2019 年的和 2026 年的项目同样有资格。github-pick 从不按创建日期预筛——年龄是事后用标签表达，绝不拿来提前排除。

A3
Reception-aware tags读得懂反响的标签

Every pick wears 🔥 trending · 🆕 new · 🏆 top-tier · 📜 has-successors. The emoji is the stable key, so the board reads the same in any language — only the label is translated.每个精选都挂着 🔥火热 · 🆕新 · 🏆一线 · 📜有继任者。emoji 是稳定的 key，所以看板在任何语言下都一致——只翻译文字。

B1
Zero build, double-click零构建，双击即开

The dashboard is a static index.html + data.js — no server, no bundler, no API key. Open it via file:// and it just works. Studied / hidden state stays local in your browser.看板就是静态的 index.html + data.js——不用服务器、不用打包器、不用 API key。file:// 双击即开。「已学 / 隐藏」状态留在你本地浏览器里。

B2
Works with any agent配任意 agent

Not a plugin and not a skill you install. An AGENTS.md / CLAUDE.md any agent reads automatically — Claude Code, Codex CLI, Cursor, Cline, Aider. The repo is the workspace.不是插件，也不是要安装的 skill。一份 AGENTS.md / CLAUDE.md，任意 agent 打开就自动读——Claude Code、Codex CLI、Cursor、Cline、Aider。这个仓库本身就是工作区。

B3
Pick a language once语言选一次

Choose a language on first run; from then on every digest and the dashboard UI stay in it. zh / en ship built-in; any other language the agent translates once. No runtime toggle, no re-asking.第一次运行选定语言；此后每一期点评和看板 UI 都用它。zh / en 自带，其他语言 agent 第一次翻译好。没有运行时切换，也不再追问。
